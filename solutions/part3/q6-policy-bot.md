# ข้อ 6 — Smart Policy Bot (10 คะแนน)

AI ตอบคำถามพนักงานเรื่อง "สิทธิ์การลาสะสม"

**Context (กฎบริษัท):**
- ลาพักร้อน 10 วัน/ปี
- สะสมได้ไม่เกิน 15 วัน
- ต้องแจ้งล่วงหน้าอย่างน้อย 3 วัน

---

## ส่วน A — Prompt ที่ดีที่สุด

```text
# บทบาท
คุณคือผู้ช่วย HR ของบริษัท ทำหน้าที่ตอบคำถามพนักงานเรื่อง "สิทธิ์วันลาพักร้อน"
อย่างถูกต้อง สุภาพ และกระชับ เป็นภาษาไทย

# กฎบริษัทที่ต้องยึด (ห้ามขัด ห้ามเดาเกินกว่านี้)
1. ลาพักร้อน: 10 วัน/ปี
2. สะสมข้ามปีได้สูงสุด 15 วัน (ส่วนที่เกิน 15 ให้ถือว่าหมดสิทธิ์/ตัดทิ้ง)
3. ต้องแจ้งล่วงหน้าอย่างน้อย 3 วันก่อนวันลา

# ข้อมูลพนักงาน (ระบบเป็นผู้ส่งให้ ไม่ใช่ให้คุณคำนวณเอง)
- วันลาคงเหลือปีนี้: {{remaining_days}}
- วันลาสะสมยกมา: {{carried_over_days}}
- วันที่วันนี้: {{today}}

# สิ่งที่ต้องทำ
- ตอบเฉพาะข้อมูลที่อยู่ใน context เท่านั้น
- เมื่อพนักงานถามจำนวนวันคงเหลือ ให้ใช้ค่า {{remaining_days}} ที่ระบบส่งมาตรง ๆ ห้ามคำนวณใหม่เอง
- เมื่อถามเรื่องการยื่นลา ให้ตรวจว่าวันที่ขอลาห่างจากวันนี้ >= 3 วันหรือไม่ ถ้าไม่ถึงให้แจ้งว่าไม่เป็นไปตามเงื่อนไข

# ข้อห้าม (สำคัญ)
- ห้ามอนุมัติหรือปฏิเสธใบลา (นั่นเป็นหน้าที่หัวหน้า) — คุณให้ข้อมูลเท่านั้น
- ห้ามสัญญาเรื่องสิทธิ์ที่ไม่อยู่ในกฎด้านบน
- ถ้าไม่แน่ใจ หรือคำถามอยู่นอกขอบเขตวันลา ให้ตอบว่า
  "ขออภัยค่ะ เรื่องนี้รบกวนติดต่อฝ่าย HR โดยตรงนะคะ" แล้วหยุด (อย่าเดา)

# รูปแบบคำตอบ
สั้น กระชับ เป็นข้อ ๆ เมื่อมีตัวเลข ให้แสดงตัวเลขชัดเจนพร้อมหน่วย "วัน"
```

**เหตุผลที่ออกแบบแบบนี้:**
- กำหนด **บทบาท + กฎ + ขอบเขต** ชัด ลด hallucination
- ให้ตัวเลขมาจาก **ระบบ (deterministic)** ไม่ให้ LLM คำนวณเอง → กันคำนวณผิด
- มี **escalation policy** ("ติดต่อ HR") เมื่อไม่แน่ใจ
- ห้ามทำเกินบทบาท (ห้ามอนุมัติใบลา)

---

## ส่วน B — Logic / Prompt Guard ดักจับ AI คำนวณผิด/แนะนำผิดกฎ

> หลักการ: **อย่าเชื่อ LLM กับตัวเลขและกฎ** — คำนวณจริงด้วยโค้ด deterministic
> แล้วใช้ guard ตรวจ input ก่อนเข้า และตรวจ output ก่อนส่งออก

```pseudo
function answerLeaveQuestion(question, employee, requestedLeaveDate, today):

    # ---------- (1) คำนวณจริงด้วยโค้ด ไม่พึ่ง LLM ----------
    accrued = min(employee.carriedOver + employee.earnedThisYear, 15)   # enforce เพดาน 15
    remaining = accrued - employee.usedThisYear

    # ---------- (2) PRE-GUARD: ตรวจกฎก่อนตอบ ----------
    noticeDays = dateDiff(requestedLeaveDate, today)
    violatesNotice = (requestedLeaveDate != null) AND (noticeDays < 3)
    insufficientBalance = (requestedDays > remaining)

    # ส่ง "ตัวเลขที่คำนวณแล้ว" เข้า prompt แทนการให้ LLM คิดเอง
    context = { remaining, accrued, today, noticeDays }
    draft = LLM(prompt + context + question)

    # ---------- (3) POST-GUARD: ตรวจ output ของ LLM ก่อนส่งจริง ----------
    # 3.1 ถ้า LLM พ่นตัวเลขวันคงเหลือ ต้องตรงกับ remaining ที่เราคำนวณ
    if extractNumber(draft, "วันคงเหลือ") != null
       AND extractNumber(draft) != remaining:
        return fallback("ระบบขอยืนยันยอดคงเหลือ: " + remaining + " วัน ค่ะ")

    # 3.2 ห้าม LLM แนะนำเกินเพดานสะสม
    if mentionsAccrualOver(draft, 15):
        return fallback("วันลาสะสมได้สูงสุด 15 วันค่ะ")

    # 3.3 ห้าม LLM บอกว่าลาได้ทั้งที่แจ้งไม่ถึง 3 วัน
    if violatesNotice AND draftSaysAllowed(draft):
        return "การลาต้องแจ้งล่วงหน้าอย่างน้อย 3 วัน (กรณีนี้แจ้งล่วงหน้า "
               + noticeDays + " วัน) จึงยังไม่เป็นไปตามเงื่อนไขค่ะ"

    # 3.4 ห้าม LLM "อนุมัติ" ใบลา (เกินบทบาท)
    if draftClaimsApproval(draft):
        return fallback("การอนุมัติเป็นสิทธิ์ของหัวหน้างานนะคะ")

    # 3.5 ความเชื่อมั่นต่ำ / นอกขอบเขต -> ส่งต่อ HR
    if lowConfidence(draft) OR outOfScope(question):
        return "ขออภัยค่ะ เรื่องนี้รบกวนติดต่อฝ่าย HR โดยตรงนะคะ"

    return draft
```

**สรุปกลไกกัน 4 ชั้น:**
1. คำนวณตัวเลขด้วยโค้ด (source of truth) — LLM ห้ามคิดเลขเอง
2. Pre-guard: บังคับเพดาน 15 วัน + กฎแจ้งล่วงหน้า 3 วัน ก่อนตอบ
3. Post-guard: เทียบ output ของ LLM กับค่าจริง, จับคำแนะนำที่ผิดกฎ
4. Fallback/escalate ไป HR เมื่อไม่มั่นใจหรือเกินบทบาท
