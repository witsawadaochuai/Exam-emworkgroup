ระบบสลับกะ & จัดการเบี้ยเลี้ยงกะดึก

เครื่องมือจำลองโครงสร้างฐานข้อมูลข้อ 5 (เปรียบเทียบพนักงานเดิม vs พนักงานปฏิบัติงานจริง)

เชื่อมต่อ PostgreSQL (localhost:5433)

กระดานข้อมูลเวรปฏิบัติงาน (Shift Assignments Board)

แสดงการมอบหมายกะในปัจจุบัน โดยแยกข้อมูลระหว่าง พนักงานเดิม (Original) และ ผู้เข้าทำงานจริง (Effective)

วันที่ กะการทำงาน เวลา พนักงานเดิมที่ได้รับมอบหมาย ผู้เข้าทำงานจริง (หลังสลับกะ) เบี้ยเลี้ยงกะดึก อ้างอิงคำขอสลับ

ส่งคำขอสลับกะปฏิบัติงาน

ยื่นคำขอเพื่อสลับกะการทำงานหรือขอให้เพื่อนร่วมงานมาทำงานแทน

ตรรกะระบบฐานข้อมูล (Database Schema Logic)

การจัดการข้อมูลและทรานแซกชันเบื้องหลัง:

1. สถานะเริ่มต้น (Initial State)

พนักงานเริ่มต้นปฏิบัติงานเองตามแผนงาน: ค่า original_emp_id และ effective_emp_id จะตรงกัน

2. สร้างคำขอสลับกะ

ระบบเพิ่มแถวใหม่ในตาราง shift_swap_requests โดยตั้งสถานะเริ่มต้นเป็น 'PENDING'

3. อนุมัติโดยหัวหน้างาน (Atomic Transaction)

เมื่อหัวหน้างานกดอนุมัติ ระบบจะรันคำสั่ง SQL ด้านล่างในชุดทรานแซกชันเดียวกัน:

UPDATE shift_swap_requests 
SET status='APPROVED', approved_by=3, approved_at=NOW() 
WHERE swap_request_id = ?;

UPDATE shift_assignments 
SET effective_emp_id = ? , swap_request_id = ? 
WHERE assignment_id = ?;

รายการรออนุมัติสลับกะ

หัวหน้างานทำการพิจารณาอนุมัติหรือปฏิเสธคำขอสลับกะ การอนุมัติจะอัปเดตเวรการทำงานจริงโดยอัตโนมัติ

เลขที่คำขอ ผู้ขอสลับ กะงานเดิมของผู้ขอ ผู้ทำงานแทน กะงานแลกคืน (ถ้ามี) เหตุผล สถานะ การจัดการ

การดึงข้อมูลเพื่อคำนวณเบี้ยเลี้ยง

คิวรี SQL ด้านล่างจะนับกะการทำงานจริงตามฟิลด์ effective_emp_id เพื่อให้จ่ายเงินเบี้ยเลี้ยงให้ผู้ที่ปฏิบัติงานกะดึกจริงได้อย่างถูกต้อง

SELECT sa.effective_emp_id              AS emp_id,
       e.full_name,
       COUNT(*)                          AS night_shifts_worked,
       SUM(s.night_allowance_satang)     AS total_allowance_satang
FROM   shift_assignments sa
JOIN   shifts    s ON s.shift_id = sa.shift_id
JOIN   employees e ON e.emp_id   = sa.effective_emp_id
WHERE  s.shift_type = 'NIGHT'
GROUP BY sa.effective_emp_id, e.full_name;

รายงานสรุปเบี้ยเลี้ยงกะดึก (จำลองผลคำนวณ)

สรุปจำนวนเวรกะดึกและยอดเบี้ยเลี้ยงกะดึกที่ได้รับของพนักงานแต่ละคน

รหัสพนักงาน ชื่อ-นามสกุล จำนวนกะดึกที่ทำงานจริง เบี้ยเลี้ยงกะดึกรวม

ระบบจำลองความสัมพันธ์และการทรานแซกชันกะการทำงาน พัฒนาขึ้นสำหรับการตรวจสอบระบบทรัพยากรบุคคล (HR Audit)
