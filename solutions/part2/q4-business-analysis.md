# ข้อ 4 — ManufacturePlus Scenario (15 คะแนน)

บริษัทผลิต 800 คน ใช้ Excel คำนวณเงินเดือน 10 ปี พบปัญหา:
สลับกะไม่เป็นระบบ / KPI ไม่ชัด / พนักงานเดินมาถาม HR เรื่องวันลา

---

## ส่วน A — Functional Requirements 3 ข้อ (ลดการเดินมาถาม HR เรื่องวันลา)

### FR-1: Employee Self-Service Leave Dashboard
หน้าจอ/แอปมือถือให้พนักงานดู **ยอดวันลาคงเหลือแบบเรียลไทม์** ได้เอง 24 ชม.
- แสดง: ลาพักร้อนได้ทั้งหมด / ใช้ไปแล้ว / คงเหลือ / สะสม / วันหมดอายุ แยกตามประเภทลา
- ดึงข้อมูลจากระบบกลาง → ไม่ต้องเดินมาถาม HR

### FR-2: Online Leave Request + Auto Balance Calculation
ยื่นใบลาออนไลน์ ระบบ **คำนวณและตัดยอดอัตโนมัติ** พร้อมตรวจสิทธิ์ก่อนส่ง
- ตรวจว่ายอดคงเหลือพอหรือไม่ / แจ้งล่วงหน้าครบ 3 วันหรือไม่ ก่อนกดส่ง
- ส่งเข้าสายอนุมัติของหัวหน้า + แจ้งผลกลับอัตโนมัติ → ไม่ต้องตามถาม HR ว่าอนุมัติยัง

### FR-3: Automated Accrual Engine + Proactive Notifications
เครื่องคำนวณการสะสมวันลาอัตโนมัติ + **แจ้งเตือนเชิงรุก**
- คำนวณสิทธิ์สะสม (เพดาน 15 วัน) ให้อัตโนมัติทุกงวด
- เตือนล่วงหน้าเมื่อ "วันลาใกล้หมดอายุ" หรือ "สะสมใกล้ชนเพดาน" → พนักงานวางแผนเองได้

> ทั้ง 3 ข้อย้ายภาระจาก "เดินมาถาม HR" ไปเป็น "ระบบตอบให้เอง + แจ้งเตือนเชิงรุก"

## ส่วน B — เทคนิค Data Cleansing (ชื่อพิมพ์ผิด / Employee ID ซ้ำ)

### 1) Normalization (ทำให้เป็นมาตรฐานก่อน)
- Trim ช่องว่างหัวท้าย, ปรับตัวพิมพ์, ตัดคำนำหน้า (นาย/นาง/Mr.), normalize Unicode
- จัดรูปแบบเบอร์โทร/เลขบัตร ปชช. ให้เป็นรูปแบบเดียว เพื่อให้เทียบกันได้

### 2) Fuzzy Matching ตรวจชื่อพิมพ์ผิด/ซ้ำ
- ใช้ **Levenshtein distance** (ระยะแก้ไข) จับชื่อที่ใกล้กัน เช่น "สมชาย" vs "สมช่าย " / "Somchai" vs "Somchia"

### 3) Deduplication + Golden Record (Employee ID ซ้ำ)
- ตรวจหา ID ซ้ำ → รวมเป็น **golden record** (เลือกค่าที่ถูกต้องที่สุดจากแต่ละแหล่ง)
- ออก **surrogate key ใหม่ที่ unique** เป็น primary key แทนการพึ่ง ID เดิมที่ซ้ำ
- เก็บ **mapping table** (old_id → new_id) ไว้ trace ประวัติเก่า ไม่ทำข้อมูลหาย

### 4) Validation Rules + Master Data Management
- ตั้งกฎตอน import: ID ต้อง unique, ฟิลด์บังคับห้ามว่าง, รูปแบบวันที่ถูกต้อง
- เคสที่ระบบตัดสินไม่ได้ → เข้าคิว **manual review** ให้ HR ยืนยัน (human-in-the-loop)
- ตั้ง **single source of truth** หลัง migrate เสร็จ เพื่อไม่ให้ Excel แตกหลายเวอร์ชันอีก
